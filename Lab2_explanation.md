## 1. Конфигурация CONFIG — что за что отвечает

```python
CONFIG = {
    "seed": 42,
    ...
}
```

### `seed: 42`
**Что это:** зерно генератора случайных чисел.  
Устанавливается одинаковым для `random`, `numpy`, `torch` и CUDA. Гарантирует **воспроизводимость** — при повторном запуске с тем же seed вы получите точно такие же результаты. Число 42 — традиционное в ML, само по себе ничего не значит.

### `data`

| Параметр | Значение | Смысл |
|---|---|---|
| `features` | 4 признака | Все столбцы, которые подаются в модель |
| `target` | `min_temp_c` | Целевая переменная — то, что модель учится восстанавливать |

- `min_temp_c` — минимальная температура за сутки (°C) ← **цель**

### `window`

| Параметр | Значение | Смысл |
|---|---|---|
| `size` | 32 | Длина одного окна — 32 последовательных дня |
| `stride` | 1 | Шаг сдвига окна — каждое следующее окно сдвинуто на 1 день |

**Что это такое:** вместо того чтобы подавать в модель одну точку, мы подаём **кусок временного ряда** — 32 дня подряд. Окна перекрываются (stride=1), поэтому из N дней получается ~N окон. Размер 32 — достаточно, чтобы уловить месячные паттерны температуры.

### `cvae` — параметры Conditional VAE

| Параметр | Значение | Смысл |
|---|---|---|
| `latent_dim` | 8 | Размерность латентного вектора **z** |
| `hidden_dims` | (256, 128, 64) | Количество нейронов в скрытых слоях энкодера и декодера |
| `lr` | 3e-4 | Скорость обучения (learning rate) для оптимизатора Adam |
| `epochs` | 200 | Количество полных проходов по обучающей выборке |
| `beta_max` | 0.02 | Максимальный вес KL-члена в функции потерь |
| `warmup_epochs` | 20 | Количество эпох для плавного увеличения β от 0 до beta_max |
| `beta_values` | [0.001...1.0] | Значения β для отдельного эксперимента по подбору β |

**`latent_dim = 8`:** весь временной ряд из 32 дней × 4 признака = 128 чисел сжимается в вектор из **8 чисел**. Это степень сжатия информации.

**`hidden_dims = (256, 128, 64)`:** архитектура нейросети воронкой — слои сужаются от 256 нейронов к 64. Энкодер: вход → 256 → 128 → 64 → z(8). Декодер — зеркально.

**`lr = 3e-4`:** стандартное значение для Adam в большинстве задач. Слишком большой lr → модель «прыгает» и не сходится; слишком маленький → учится медленно.

**`beta_max = 0.02`:** очень маленькое значение. Это означает, что реконструкции уделяется значительно больше веса, чем регуляризации латентного пространства. Выбор обоснован — у нас задача восстановления, а не генерации.

**`warmup_epochs = 20`:** β линейно нарастает от 0 до 0.02 за первые 20 эпох. Если сразу включить KL-штраф, модель может "схлопнуть" латентное пространство в начале обучения (posterior collapse).

---

### `vqvae` — параметры Conditional VQ-VAE

| Параметр | Значение | Смысл |
|---|---|---|
| `embedding_dim` | 8 | Размерность каждого вектора в кодбуке |
| `num_embeddings` | 32 | Количество векторов в кодбуке (размер "словаря") |
| `num_latent_tokens` | 4 | На сколько дискретных токенов кодируется одно окно |
| `commitment_beta` | 0.25 | Вес commitment loss — штраф за "отдаление" энкодера от кодбука |
| `lr` | 1e-3 | Скорость обучения |
| `epochs` | 200 | Эпохи обучения |
| `hidden_dims` | (256, 128, 64) | Слои нейросети (аналогично CVAE) |

**`num_embeddings = 32`:** 32 прообраза-паттерна погоды. Каждое окно кодируется как набор из 4 ближайших прообразов. Можно считать это "словарём типичных погодных ситуаций".

**`num_latent_tokens = 4`:** окно из 32 дней сжимается в **4 дискретных токена**. Каждый токен — это индекс одного из 32 векторов в кодбуке.

**`commitment_beta = 0.25`:** если энкодер начинает "убегать" от ближайшего вектора кодбука, этот штраф его возвращает. Значение 0.25 — стандартное из оригинальной статьи VQ-VAE.

## 2. Определения ключевых понятий

### VAE (Variational Autoencoder)
Нейросеть, которая **сжимает** данные в маленькое латентное пространство (энкодер), а затем **восстанавливает** их обратно (декодер). В отличие от обычного автоэнкодера, латентный вектор **z** — не просто точка, а **распределение** (μ и σ), из которого семплируется z ~ N(μ, σ²). Это позволяет генерировать новые данные.

### VQ-VAE (Vector Quantized VAE)
Вариация VAE, где латентное пространство **дискретное**. Вместо непрерывного вектора z энкодер выдаёт вектор, который **заменяется на ближайший вектор из кодбука** (словаря). Аналогия: вместо "этот день чуть холоднее обычного" — "этот день соответствует паттерну №17 из словаря".

### Функция потерь CVAE: ELBO
```
Loss = Reconstruction Loss + β × KL Divergence
```
- **Reconstruction Loss (MSE):** насколько точно декодер восстановил исходное окно
- **KL Divergence:** насколько распределение q(z|x) отличается от N(0,1). Это регуляризация — не даёт латентному пространству стать произвольным
- **β:** вес компромисса между точностью и регулярностью

### Функция потерь VQ-VAE
```
Loss = Reconstruction Loss + Codebook Loss + β_commitment × Commitment Loss
```
- **Codebook Loss:** двигает векторы кодбука к энкодерным представлениям
- **Commitment Loss:** двигает энкодер к ближайшему вектору кодбука

### Posterior Collapse
Патология VAE: когда β слишком большой, KL-штраф заставляет q(z|x) → N(0,1), и **z перестаёт нести информацию о входе**. Декодер игнорирует z и учится восстанавливать среднее по датасету. Именно поэтому используется beta_max=0.02 и warmup.

### Perplexity кодбука (VQ-VAE)
Показывает, сколько векторей кодбука реально используется. Формула: `exp(-Σ p_i × log(p_i))`. При 32 кодах максимальная perplexity = 32 (все коды используются равномерно). Низкая perplexity → кодбук не используется эффективно (codebook collapse).

### StandardScaler
Нормализация: `x_scaled = (x - mean) / std`. Приводит все признаки к нулевому среднему и единичному стандартному отклонению. Необходим, потому что осадки (0–50 мм) и температура (−40...+40°C) имеют разные масштабы.

Три цветных отрезка по оси времени. Подтверждает, что разбивка **хронологическая**: train — первые 70% дат, val — следующие 15%, test — последние 15%.

### График 3: Предпросмотр окон
Несколько примеров окон размером 32. Показывает, как выглядит один обучающий пример — кусок 32-дневного временного ряда всех 4 признаков.

### Графики обучения CVAE (4 панели)

| Панель | Что показывает | На что смотреть |
|---|---|---|
| Total Loss | Полная функция потерь (train vs val) | Должна монотонно убывать; расхождение train/val = переобучение |
| Reconstruction Loss | MSE восстановления окна | Основной критерий качества |
| KL Loss | Расхождение Kullback–Leibler | До warmup=0, потом растёт до стабильного значения |
| Beta Schedule | График β по эпохам | Линейный рост 0→0.02 за 20 эпох, затем константа |

**Ваши значения:** val reconstruction loss стабилизировался — признак, что модель сошлась. KL удерживается на небольшом значении благодаря малому beta_max=0.02.

### Графики обучения VQ-VAE (5 панелей)

| Панель | Что показывает |
|---|---|
| Total Loss | Сумма всех потерь |
| Reconstruction Loss | MSE |
| Codebook Loss | Насколько далеко кодбук от энкодерных представлений |
| Commitment Loss | Насколько далеко энкодер от ближайшего вектора кодбука |
| Perplexity | Эффективное использование кодбука |

**Perplexity у вас:** если близка к 32 — все коды задействованы, кодбук богатый. Если 5–10 — большинство окон кодируется несколькими паттернами, кодбук "схлопнулся".

### Графики реконструкции (примеры окон)
Для каждого из 5 примеров: синяя линия — исходное окно, оранжевая — восстановленное моделью. Чем ближе они друг к другу, тем лучше. Обратите внимание: модель обычно хорошо ловит тренд, но сглаживает острые пики — это нормально для регрессии через VAE.

### Scatter plot: предсказанное vs реальное
Точки — пары (реальное min_temp_c, предсказанное min_temp_c). Идеальная модель → все точки на диагонали y=x. Отклонение от диагонали = ошибка. **R²=0.94** означает, что точки плотно обнимают диагональ.

### График ошибок (residuals)
Распределение ошибок (реальное − предсказанное). Должно быть симметрично вокруг нуля. Если смещено — у модели систематическая ошибка (bias). **MAE=2.14–2.38°C** — в среднем модель ошибается примерно на 2°C.

### Эксперимент с Beta (6 панелей для разных β)
Сравниваются кривые обучения для β ∈ {0.001, 0.01, 0.05, 0.1, 0.5, 1.0}.

**Что смотреть:**
- **Recon loss:** растёт с ростом β — регуляризация мешает реконструкции
- **KL loss:** падает с ростом β — модель "прячет" меньше информации в z
- **Val MSE:** минимум около β=0.001–0.01; ваш рабочий beta_max=0.02 находится в оптимальной зоне

### График использования кодов VQ-VAE (bar charts)
Гистограмма: по оси X — номер кода (0–31), по оси Y — сколько раз он был выбран. Равномерное распределение — хороший знак. Доминирование 2–3 кодов — codebook collapse.

### Финальный "сшитый" ряд (overlap-add)
Все предсказанные окна накладываются обратно на временную ось с усреднением перекрытий. Получается полный восстановленный временной ряд. Сравнивается с реальным. **Если кривые практически совпадают — модель успешно усвоила сезонную структуру.**

**R²=0.94** — очень хороший результат для прогноза погоды. Для сравнения:
- R² < 0.7 — слабая модель
- R² 0.7–0.9 — приемлемо
- R² > 0.9 — хорошо
- R² > 0.95 — отлично (часто подозрительно хорошо — возможна утечка данных)

**MAE≈2°C** при диапазоне min_temp_c ≈ −40...+25°C (65°C) означает относительную ошибку ~3% — хороший результат.

**VQ-VAE лучше CVAE** несмотря на то, что работает с дискретными кодами — это говорит о том, что погодные паттерны действительно хорошо описываются небольшим набором прообразов (32 шт.).
